# Contributing

## Branching
- `main` is protected. Create feature branches from issues.

## Commits
- Conventional commits: `feat:`, `fix:`, `docs:`, `chore:`, `test:`, `refactor:`.

## Tests
- Add tests with each change (frontend: vitest / testing-library if enabled).

## GAS Deploy
- Use `clasp version` + `clasp deploy` for stable releases only.
